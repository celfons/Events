Architecture Highlights

  • Clean Architecture: Separation of concerns with domain, application, and infrastructure layers
  • SOLID Principles: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion
  • Clean Code: Readable, maintainable, and self-documenting code
  • Repository Pattern: Abstraction of data access logic
  • Dependency Injection: Loose coupling and better testability

API Endpoints

Events:
GET /api/events - List all events
GET /api/events/:id - Get event details
POST /api/events - Create new event

Registrations:
POST /api/registrations - Register for event
POST /api/registrations/:id/cancel - Cancel registration

Project Structure

src/
├── domain/              # Business entities & interfaces
│   ├── entities/        # Event, Registration
│   └── repositories/    # Repository interfaces
├── application/         # Use cases (business logic)
│   └── use-cases/       # CreateEvent, RegisterForEvent, etc.
├── infrastructure/      # External concerns
│   ├── database/        # MongoDB implementation
│   └── web/             # Express controllers & routes
└── server.js            # Application entry point

Quick Start

  1. Clone the repository
  2. Run npm install
  3. Configure MongoDB connection in .env
  4. Run npm run seed to populate sample data
  5. Run npm start to start the server
  6. Open http://localhost:3000 in your browser
